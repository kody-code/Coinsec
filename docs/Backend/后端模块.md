# 个人记账软件（CoinSec）后端完整设计

**技术栈:**

- **框架:** Spring Boot 4.0.6
- **语言:** Java 25
- **数据库:** PostgreSQL（pgsql）
- **ORM:** Spring Data JPA
- **权限:** Sa-Token
- **构建:** Gradle Kotlin

## 一、后端模块设计（共 8 大核心模块）

1. 用户模块（User Module）
   负责：登录、注册、个人信息、头像上传、密码修改
   对应表：`users`
2. 分类模块（Category Module）
   负责：收入 / 支出分类管理、图标、排序
   对应表：`categories`
3. 账户模块（Account Module）
   负责：资金账户（微信 / 支付宝 / 现金）、余额、图标
   对应表：`accounts`
4. 记账记录模块（Record Module）
   负责：新增收支、查询流水、筛选、统计
   对应表：`records`（核心）
5. 转账模块（Transfer Module）
   负责：账户之间互转、余额自动变更
   对应表：`transfers`
6. 权限认证模块（Auth Module）
   负责：登录状态、token 校验、接口保护
   技术：`Sa-Token`
7. 通用工具模块（Common Module）
   负责：统一返回结果、时间工具、异常处理、文件工具
8. 配置模块（Config Module）
   负责：数据库、跨域、Sa-Token、全局配置

## 二、后端文件结构设计（标准 Spring Boot 结构）

```text
com.kody.coinsec
├── config          # 配置类
├── controller      # 控制器（API接口）
├── entity          # JPA实体（对应pgsql表）
├── repository      # 数据访问（JPA）
├── service         # 业务逻辑
│   ├── impl        # 业务实现
├── common          # 公共模块
│   ├── result      # 统一返回格式
│   ├── exception   # 全局异常
├── util            # 工具类
└── CoinsecApplication.java  # 启动类
```

## 三、各包职责

1. config

- 跨域配置
- Sa-Token 配置
- 数据库 / JPA 配置
- 全局异常配置

2. controller

- UserController
- CategoryController
- AccountController
- RecordController
- TransferController
- AuthController

3. entity（对应 PostgreSQL 表）

- UserEntity（用户）
- CategoryEntity（分类）
- AccountEntity（账户）
- RecordEntity（记账记录）
- TransferEntity（转账）

4. repository
   数据访问层，继承 JpaRepository，操作 PostgreSQL
5. service
   业务逻辑层，处理记账、统计、余额变更、业务规则
6. common
   全局统一响应、全局异常处理、常量定义
7. util
   工具类：时间、文件、字符串、加密等

## 五、API 接口设计

- 用户：登录 / 注册 / 获取信息 / 修改头像
- 分类：增删改查
- 账户：增删改查 / 获取余额
- 记录：新增收支 / 按月查询 / 统计
- 转账：账户互转 / 记录查询
