# 个人记账软件数据库（PGSQL）

## 一、数据库总设计

- 数据库名：`coinsec_db`
- 核心表：5 张（满足个人记账全部场景）
- 设计原则：轻量、易维护、支持收支统计、分类管理、备注、时间筛选

## 二、数据表设计（5 张核心表）

1. 用户表（users）
   个人使用只需 1 条数据，用于登录、基础信息
   | 字段 | 类型 | 约束 | 说明 |
   |-------------|--------------|--------|--------------|
   | user_id | SERIAL | 主键 | 用户唯一标识 |
   | username | VARCHAR(50) | 非空 | 登录账号 |
   | password | VARCHAR(100) | 非空 | 加密密码 |
   | nickname | VARCHAR(50) | - | 昵称 |
   | avatar | VARCHAR(255) | - | 头像图片路径 / URL |
   | create_time | TIMESTAMP | 默认当前时间 | 账号创建时间 |

2. 分类表（categories）
   管理收入 / 支出分类，如：餐饮、交通、工资、红包等
   | 字段 | 类型 | 约束 | 说明 |
   |-------------|--------------|------------|-----------------------|
   | category_id | SERIAL | 主键 | 分类 ID |
   | user_id | INT | 外键关联 users | 归属用户 |
   | name | VARCHAR(30) | 非空 | 分类名称 |
   | type | VARCHAR(10) | 非空 | income 收入 /expense 支出 |
   | icon | VARCHAR(255) | - | 分类图标地址 |
   | sort | INT | 默认 0 | 排序权重 |

3. 账户表（accounts）
   管理资金来源：微信、支付宝、现金、银行卡等
   | 字段 | 类型 | 约束 | 说明 |
   |------------|---------------|------------|-----------|
   | account_id | SERIAL | 主键 | 账户 ID |
   | user_id | INT | 外键关联 users | 归属用户 |
   | name | VARCHAR(30) | 非空 | 账户名称 |
   | icon | VARCHAR(255) | - | 账户图标地址 |
   | balance | DECIMAL(12,2) | 默认 0.00 | 当前余额 |
   | status | INT | 默认 1 | 1 正常 0 停用 |

4. 记账记录表（records）【核心主表】
   每一笔收入 / 支出都存在这里
   | 字段 | 类型 | 约束 | 说明 |
   |-------------|---------------|-----------------|-----------|
   | record_id | SERIAL | 主键 | 流水 ID |
   | user_id | INT | 外键关联 users | 归属用户 |
   | category_id | INT | 外键关联 categories | 所属分类 |
   | account_id | INT | 外键关联 accounts | 扣款 / 入账账户 |
   | type | VARCHAR(10) | 非空 | 收支类型 |
   | amount | DECIMAL(12,2) | 非空 | 交易金额 |

5. 转账记录表（transfers）
   用于账户之间互转：微信 → 支付宝
   | 字段 | 类型 | 约束 | 说明 |
   |-----------------|---------------|---------------|-------|
   | transfer_id | SERIAL | 主键 | 转账 ID |
   | user_id | INT | 外键关联 users | 归属用户 |
   | from_account_id | INT | 外键关联 accounts | 转出账户 |
   | to_account_id | INT | 外键关联 accounts | 转入账户 |
   | amount | DECIMAL(12,2) | 非空 | 转账金额 |
   | transfer_time | TIMESTAMP | 非空 | 转账时间 |

## 三、数据库设计亮点

1. 结构极简：仅 5 张表，个人使用完全足够，无冗余
2. 支持完整场景
   - 收入 / 支出记账
   - 多账户管理
   - 分类统计
   - 转账记录
   - 时间筛选、金额统计
3. PostgreSQL 原生适配
   - 支持 DECIMAL 精确金额
   - 支持 TIMESTAMP 时间索引
   - 支持统计类 SQL（月账单、年账单）
4. 易扩展: 可后续加：标签、预算、报表、图片附件

## 四、业务逻辑说明

1. 用户：唯一操作者，管理自己的账单
2. 分类：区分收入 / 支出，用于统计开销结构
3. 账户：管理资金载体，记账时自动增减余额
4. 记录：核心流水，每笔消费 / 收入都生成一条记录
5. 转账：账户间资金转移，不影响总资产，只改变账户余额
